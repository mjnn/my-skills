# 账号 APP 埋点设计参考案例

> Read when: 需要了解完整设计规模与模块覆盖时

## v1 基准（仅 SPEC）

- 25 个事件、6 条路径、3 个漏斗、10 条关联
- 模块：用户认证、个人信息、车辆绑定、消息通知

## v3 UE 校验版（SPEC + UE Design V0.1_26KW22 + SPEC v0.10）

- 23 个事件、6 条路径、14 条漏斗步骤、10 条关联
- 覆盖 UE 全部 10 个功能模块：
  1. 启动应用
  2. 车辆绑定
  3. 账号登录
  4. Face ID 登录
  5. 账号管理
  6. FacialD 管理
  7. 三方账号管理
  8. 流量管理
  9. 激活引导
  10. 隐私协议 / Token 失效

## 典型事件示例

| 事件ID | 事件名称 | 类型 |
|--------|---------|------|
| ACC_001 | 账号登录 | 状态变更 |
| ACC_101 | 个人资料查看 | 页面浏览 |
| ACC_201 | 车辆绑定开始 | 流程节点 |
| ACC_501 | 消息接收 | 操作交互 |

## 典型路径示例

| 路径ID | 路径名称 | 关键节点 |
|--------|---------|---------|
| PATH_001 | 首次使用路径 | 注册→绑定→首页 |
| PATH_002 | 登录恢复路径 | 登录失败→重试→成功 |
