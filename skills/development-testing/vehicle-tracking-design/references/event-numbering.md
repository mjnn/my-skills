# 事件编号与类型规范

> Read when: Step 2 分析 SPEC 并生成事件时

## 事件类型分类

| 类型 | 事件模式 | 示例 |
|------|---------|------|
| 页面浏览 | 进入页面、离开页面 | ACC_101(个人资料查看) |
| 操作交互 | 按钮点击、开关切换、输入提交 | ACC_102(个人资料编辑) |
| 流程节点 | 流程开始、步骤完成、流程结束 | ACC_201(车辆绑定开始) |
| 状态变更 | 登录/登出、绑定/解绑 | ACC_004(退出登录) |
| 异常事件 | 操作失败、超时、错误 | 关联到 REL_009(异常→重试) |

## 模块编号分段

| 模块编号 | 模块类型 | 适用场景 |
|----------|---------|---------|
| 001-099 | 用户认证类 | 登录、注册、注销、密码 |
| 100-199 | 个人信息类 | 资料查看、编辑、头像 |
| 200-299 | 车辆控制类 | 车辆绑定、解绑、状态控制 |
| 300-399 | 服务功能类 | 预约、保养、道路救援 |
| 400-499 | 商城/积分类 | 积分查询、兑换、任务 |
| 500-599 | 消息通知类 | 消息接收、点击、设置 |
| 600-699 | 设置/隐私类 | 隐私设置、偏好修改 |
| 700-799 | 导航类 | 导航开始、结束、POI搜索 |
| 800-899 | 媒体/娱乐类 | 音乐、视频、电台 |
| 900-999 | 系统级类 | 系统设置、OTA、诊断 |

## ID 格式

| 类型 | 格式 | 示例 |
|------|------|------|
| 事件 | `[APP前缀]_[模块编号]` | ACC_001, NAV_305 |
| 路径 | `PATH_[序号]` | PATH_001 |
| 漏斗 | `FUNNEL_[序号]` | FUNNEL_001 |
| 关联 | `REL_[序号]` | REL_001 |

## 路径 / 漏斗 / 关联设计要点

**用户路径**：首次使用路径、核心功能路径、异常恢复路径、跨模块路径。

**转化漏斗**：每个关键业务流程拆解为 3-6 个步骤节点，每步定义转化指标和优化目标。

**关联埋点**：消息推送→功能使用、页面跳转→功能转化、功能完成→后续行为、异常→重试/退出/客服。
