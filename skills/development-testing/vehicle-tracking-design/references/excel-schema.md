# 输出 Excel 规范

> Read when: Step 4 生成 Excel 或向用户汇报交付内容时

OOXML 生成顺序与损坏排查见 `references/xlsx-generation.md`。

## Sheet 1: [APP名称]埋点设计

| 列名 | 说明 |
|------|------|
| 事件ID | 唯一事件编号（如 ACC_001） |
| 事件名称 | 中文事件名 |
| 事件说明 | 触发场景描述 |
| 分析目的 | 为什么采集此事件 |
| 核心指标 | 关键度量指标 |
| 上报参数 | 逗号分隔的参数列表 |
| 参数说明 | 对应参数的中文说明 |
| 数据分类 | 数据分类标签 |

## Sheet 2: 用户路径埋点

| 列名 | 说明 |
|------|------|
| 路径ID | 唯一路径编号 |
| 路径名称 | 路径中文名 |
| 路径描述 | 完整路径描述 |
| 关键节点 | 节点序列 |
| 转化目标 | 目标转化率 |
| 埋点事件 | 关联的事件ID列表 |
| 分析指标 | 路径分析指标 |
| 数据分类 | 数据分类标签 |

## Sheet 3: 转化漏斗埋点

| 列名 | 说明 |
|------|------|
| 漏斗ID | 唯一漏斗编号 |
| 漏斗名称 | 漏斗中文名 |
| 漏斗步骤 | 步骤编号和名称 |
| 步骤描述 | 步骤行为描述 |
| 埋点事件 | 对应事件ID |
| 转化指标 | 该步骤的转化指标 |
| 优化目标 | 优化方向 |
| 数据分类 | 数据分类标签 |

## Sheet 4: 关联埋点设计

| 列名 | 说明 |
|------|------|
| 关联ID | 唯一关联编号 |
| 关联类型 | 关系类型（消息→功能、首页→功能等） |
| 源事件 | 起始事件 |
| 目标事件 | 目标事件 |
| 关联关系 | 关系描述 |
| 埋点参数 | 关联分析用参数 |
| 分析目的 | 关联分析目的 |
| 数据分类 | 数据分类标签 |
